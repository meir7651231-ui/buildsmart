## Part 0 — Orientation

The Preact app is the cutover reference for the Flutter rewrite. Its data is
**auto-generated** from the legacy `index.html` prototype by
`scripts/extract-catalog.mjs` (catalog/variants/suppliers/tools all carry the
header `Auto-generated by scripts/extract-catalog.mjs — do not edit by hand`).
The state layer is hand-written `@preact/signals` stores. The whole UI is governed
by 9 design rules (R1–R9), the most important being **R2 (no window, ever)** — and
the build history is essentially the story of learning to obey R2.

| Layer | Files | Generated? |
|---|---|---|
| Data | `src/data/{catalog,variants,suppliers,tools,identity,projects,search-index}.ts` | catalog/variants/suppliers/tools = generated; identity/projects/search-index = hand-written |
| Stores | `src/store/{app-store,app-settings,bs-store,user-profile,search-store,toast-store,regression-store}.ts` | hand-written signals |
| Lib | `src/lib/{search,barcode,voice}.ts` | hand-written |

# PART 3 — LIB (`src/lib/`)

## 3.1 `search.ts` — the search engine

Legacy: `searchSuggestions` @ `:8629-8651`, `fuzzySearchSuggest` @ `:21095-21114`.
```ts
searchExact(query, limit=40): SearchHit[]   // substring match, prefix-first
searchFuzzy(query, limit=6): SearchHit[]     // Levenshtein fallback
```
- `normalize` = `toLocaleLowerCase('he')`.
- **Exact**: scans `searchIndex()`, finds best (lowest) match position across each
  hit's keywords; buckets prefix (pos 0) vs contains; score = `pos*100 +
  KIND_ORDER[kind]` where `KIND_ORDER = {screen:0, cat:1, prod:2}` (screens rank
  first). Returns prefix bucket then contains bucket, sliced to limit.
- **Fuzzy**: only runs when exact returned nothing **and** query ≥ 2 chars.
  Tolerance = `floor(len/3)+1` (≈1 edit per 3 chars). Damerau-style Levenshtein over
  sliding windows of each keyword; scored by `dist + KIND_ORDER*0.1`.

## 3.2 `barcode.ts` — BarcodeDetector wrapper
- `isBarcodeSupported()` — needs `window.BarcodeDetector` + `getUserMedia`
  (Chromium only; Safari/Firefox lack it).
- `startBarcodeScanner(videoEl, onDetect, onError)` — opens rear camera
  (`facingMode: environment`), scans formats `ean_13, ean_8, code_128, code_39,
  qr_code` via `requestAnimationFrame` loop; returns `{video, stop}`.
- Hebrew error strings: `'הדפדפן הזה לא תומך בסריקת ברקוד'`, `'אין גישה למצלמה'`.

## 3.3 `voice.ts` — Web Speech API wrapper
- `isVoiceSupported()` — `SpeechRecognition || webkitSpeechRecognition`.
- `startVoiceRecognition({onTranscript, onEnd, onError})` — `lang='he-IL'`,
  `continuous=false`, `interimResults=true`; returns `{stop}`.
- Hebrew error: `'הדפדפן הזה לא תומך בחיפוש קולי'`.

# PART 4 — RULES (`RULES.md` — R1–R9, VERBATIM)

> `RULES.md` is the spec for **how** to present; the prototype `index.html` is the
> spec for **what** to present. On conflict: prototype provides content, rules provide
> the wrapper. Any code in `app/` violating a rule is a **bug, even if it compiles.**

**R1 · חמשת הסמלים הראשיים לא זזים. בשום מצב.**
5 fixed FABs never move/hide/resize in any state. Slots: Top-right = זהות/BS Logo
(opens persona dial), Top-center = persona name (text only, not clickable), Top-left
= cart, Bottom-left = menu FAB, Bottom-right = search FAB. Only `:active`
press-state (scale 0.92–0.95) and temporary `z-index` raises are allowed;
`bottom`/`top`/`inset` stay identical.

**R2 · אין חלון מלא. backdrop קל לסימון מצב פעיל בלבד.**
Forbidden: full-height slide-in drawer; bottom/top sheet with primary content;
modal/dialog that blocks flow; `role="dialog" aria-modal="true"` on tool menus;
heavy backdrop (>~50% opacity or >4px blur). Explicitly allowed: **product-sheet**
(the single "secondary screen" exception), and **light backdrops** (opacity 30–45%,
blur ≤3px) on the menu speed-dial + search panel — only to signal active-mode and
accept click-to-close. **The principle: when the prototype opens a full window, we
translate it to a dial.**

**R3 · Dial — הצורה היחידה לפתיחת כלים.** Tapping a primary button reveals its tools
as a **dial**. Open direction follows the button's side (top button → drops down;
bottom FABs → rise up). Guard wall = the button's side (BS/search = right; cart/menu
= left). Animation = `dial-in` keyframe (opacity 0→1, translateY 8px→0, ~30ms
stagger). Forbidden: connected list, scrim, full-screen drawer.

**R4 · פריט dial = שני אלמנטים נפרדים.** Every dial item = a `[circle 48×48]` and a
separate `[pill label, padding 6×13]` with a gap, each with its own
`background`/`box-shadow`. **Forbidden:** wrapping icon+label in one
container/pill/row with a single background (the deleted `bsrow` of `bs-panel.tsx`
was the violation). **Selected state:** both circle and pill switch together to
`background: var(--brand); color:#fff`.

**R5 · בחירת tool.** On tapping a tool in an active dial: (1) other tools vanish
immediately; (2) the selected stays in slot 1 (nearest the parent button); (3) if it
has a sub-menu it opens above (bottom button) or below (top button) in R4 style;
(4) re-tap the selected → back to the full dial; (5) re-tap the parent button →
full close.

**R6 · האב-טיפוס הוא ה-spec.** Everything in `index.html` must be implemented in
`app/`: all 5 personas, all their screens, all flows (registration/login/checkout…),
all data (TREES, VARIANTS, STORE_PRICING, TOOLS, SUPPLIER_STORES…). Forbidden:
cutting features without explicit consent; inventing content not in the prototype
(e.g. the "dark mode / language" once wrongly added to BS); deciding "contractor
only" because "it's a prototype" — the prototype is the spec, not the MVP.

**R7 · אסור להמציא תוכן.** Before adding any feature/text/option/button, verify it
exists in `index.html`. If not — must ask first.

**R8 · RTL — בית/חיפוש ימינה, חנות/עגלה שמאלה.** Identity/search actions align right
(`inset-inline-start` in RTL); commerce/cart/store actions align left
(`inset-inline-end`).

**R9 · שדות טקסט = שורת הקלדה inline, צמודה לעלה.** Free-text settings leaves (name,
phone, business type, trade, payment…) **do not** open `prompt()`, sheet, modal, or
drawer. Behavior: tap → input row opens **adjacent to the leaf**; the row is the same
two R4 elements but with `<input class="dial__input">` replacing
`<span class="dial__label">`; input autofocus, value=current, `dir=auto`; **save** on
`Enter`/blur (writes store + toast `"<field> עודכן"`, closes row); **cancel** on
`Esc` (no save); the dial **stays open** throughout. Empty value = no
`dial__circle--on`.

> `RULES.md` also embeds the **Building Inspector Protocol** (checkpoints by stage,
> CRITICAL/MAJOR/MINOR severity with blocking authority, code-loop + stuck-loop
> detection) and the **Reporting Protocol** (Hebrew summary format) — both detailed
> in Part 6.

# PART 5 — ADRs

## ADR-001 · No-Window UI (`adr/001-no-window.md`)
**Status:** Accepted (2026-05-20). The human-readable form is R2.
- **Context:** the legacy prototype opens drawers/modals/sheets liberally;
  construction-site users (dirty hands, small screens, single-handed, interrupted)
  break flow on every modal. Owner stated repeatedly: *"אף אחד מהם לא פותח חלון.
  נקודה."* Claude's first window-opening features (`bs-panel.tsx` drawer, search
  submenus as sheets) had to be rebuilt.
- **Decision:** no full-window overlay. No side drawer, no bottom/top sheet, no
  `role=dialog aria-modal` on tool menus, no backdrop beyond ≤45% opacity / ≤3px
  blur. Two bounded exceptions: `product-sheet`, and the light dial/search
  backdrops. When legacy opens a window → translate to a **dial**.
- **Rationale:** speed (no close gesture first), low cognitive load (nothing hidden),
  single-hand operation (dials near the FAB), recovery (re-tap to reverse).
- **Verification:** Inspector frame FRM-02 flags new `position:fixed; inset:0`;
  finish FRM-06 checks backdrop opacity/blur.

## ADR-002 · The Dial Pattern (`adr/002-dial-pattern.md`)
**Status:** Accepted (2026-05-20). Human-readable form is R3/R4/R5.
- **Context:** once ADR-001 banned windows, an alternative was needed for revealing a
  button's tools. The dial: a vertical column of compact buttons dropping below /
  rising above the primary control on the same side.
- **Decision:** same-side anchoring (BS top-right → drop down right; search
  bottom-right → rise up right; menu bottom-left → rise up left; cart top-left → drop
  down left); two separate elements per item (circle + pill, ~10px gap); active item
  highlights both in teal; sub-menu collapses siblings, keeps selected in slot 1,
  opens in the same two-element style.
- **Rationale:** visual consistency with the no-window rule; the gap lets the
  bathroom background show through (part of the look); thumb reachability.
- **Alternatives rejected:** single-chip items (too heavy, hides background);
  side-drawer w/icons (ADR-001); bottom sheet (ADR-001); horizontal dial (less
  reachable). Implementers: `bs-dial.tsx`, `tools-dial.tsx` (search),
  `menu-speed-dial.tsx`.

> `adr/README.md` defines when an ADR is required (rule change, non-obvious pattern,
> persistent tech decision, accepted violation) and the numbering rule (zero-padded,
> never reused; obsolete ADRs marked `Superseded by`, file kept).

# PART 6 — INSPECTOR PROTOCOL (`knowledge/inspector/`)

The Inspector replaces the old PASS/FAIL chat audit. It is an **independent Explore
subagent**, run before every commit, structured by 5 build stages, with severity
levels and blocking authority. Reports are immutable and written to
`inspections/INSP-NNNN-*.md`.

### Stages & triggers (`README.md` / `RULES.md`)
| Stage | Runs when the diff touches |
|---|---|
| **foundation** | `src/store/`, `src/data/`, `src/test/types.ts` |
| **frame** | `src/components/`, `src/views/`, `src/app.tsx` |
| **wiring** | new handlers / signal mutations / effects |
| **finish** | `src/styles/`, classes, ARIA |
| **operations** | always — last stage before commit |

### Severity → consequence
| Severity | Action |
|---|---|
| **CRITICAL** | commit auto-blocked; fix or write an ADR |
| **MAJOR** | needs owner approval; cannot proceed without |
| **MINOR** | documented only; non-blocking |

### Checklist highlights (`checklist.md`)
- **Foundation:** FND-01 typecheck; FND-02/03 no duplicate IDs in PRODUCTS/CATEGORIES;
  FND-05 data arrays non-empty; **FND-06** every `STORE_PRICING` SKU exists in
  `VARIANTS.opts[].sku`; FND-07 every VARIANTS key is a real product; FND-09
  localStorage `bs.{thing}.v{N}`.
- **Frame:** FRM-01 (R1) FABs fixed; **FRM-02 (R2)** no new `position:fixed; inset:0`
  overlay; FRM-03 (R3) dial not list/drawer; FRM-04 (R4) separate circle+label;
  FRM-06 backdrop opacity ≤0.45 / blur ≤3px; FRM-07 persona views render (testTabs
  PASS).
- **Wiring:** **WIR-02** `cartCount === Σ cart.qty` invariant; WIR-03/04/05 no infinite
  effect loops / signal mutation in same-signal effect / setter in render body.
- **Finish:** FIN-01 (R8) RTL sides; FIN-03 Hebrew `aria-label`; FIN-05 touch ≥44×44.
- **Operations:** OPS-01 typecheck, OPS-02 build, OPS-03 regression no FAIL, **OPS-06
  process-loop** (same finding ID not in 2+ of last 3 reports).

### Loop detection (`loops.md`)
- **Code loops L-01…L-08** (grep patterns): L-02 setter for a value in its own effect
  deps (CRITICAL); L-03 signal mutation inside an `effect()` that reads the same
  signal (CRITICAL); L-04/L-05 setter / `.value=` in render body (CRITICAL); L-07
  `while(true)`/`for(;;)`/unbounded recursion (CRITICAL).
- **Process loop P-01 (stuck-loop):** if the same finding ID appears in ≥2 of the last
  3 reports → promote to CRITICAL, tag `stuck-loop`, emit
  `המפקח חוזר על אותו ממצא — חייבת התערבות הבעלים, לא ניסיון נוסף`, return
  `VERDICT: NO-GO (stuck loop)`, and **stop**. Claude must stop fixing and ask the
  owner. P-02: 5+ inspections for the same stage in 10 min = thrashing (MAJOR).

### Prompt (`prompt.md`)
Verbatim master prompt with `{STAGE}` placeholder; the subagent reads RULES.md +
checklist + loops + last 3 reports, examines `git diff HEAD~1 -- app/src`, applies
only its stage's items, and returns a fixed-format report ending in the VERDICT line.
"The report **is** the output" — no editorializing.

### Reporting protocol (`reporting.md`)
After every significant action Claude must produce a fixed Hebrew summary with 7
mandatory sections: **מה עשיתי · איך הלך** (VERDICT + severity counts) **· שינויים
שאתה רואה** (≤3 bullets) **· סיכונים** (never "no risk" if one exists) **· מצב נוכחי**
(numbers) **· מה הלאה** (1–3 options) **· צריך החלטה ממך?**. Rules: simple Hebrew,
numbers not vagueness, ≤200 words, always end with a question/offer.

# → FLUTTER NOTES

**Mirror exactly (frozen content — verbatim per R6/R7/R8):**
- **All data files** as-is: `catalog.ts` (40 cats / 202 products / 53 with
  accessories, the `must`/`why` accessory model), `variants.ts` (44 defs), `suppliers.ts`
  (3 suppliers, 696 prices), `tools.ts` (21 bundles), `identity.ts` (4 RANKS / 6
  achievements / formatIls), `projects.ts` (3 projects), `search-index.ts` (7 screens +
  40 cats + 202 products). All Hebrew strings, emoji, and numeric values are
  load-bearing and must be byte-for-byte. Ideally regenerate from the same
  `index.html` rather than hand-transcribe.
- **The search engine** (`search.ts`): prefix-first exact + Levenshtein fuzzy
  (tolerance `floor(len/3)+1`, fuzzy only when exact empty), `KIND_ORDER`
  screen<cat<prod. Port the algorithm faithfully; Hebrew `toLocaleLowerCase` matters.
- **Persistence keys** (`bs.settings.v1`, `bs.profile.v1`, `bs.persona.v1`,
  `bs.search.recent.v1`) and the `bs.{thing}.v{N}` convention — keep identical for
  data continuity if any migration is desired.
- **The store shapes & invariants:** AppSettings DEFAULTS, the cart `cartCount === Σ
  qty` invariant, `editingLeafKey` reset-on-every-transition (R9), `bsDrillPersona`
  must NOT change the active persona.
- **The 9 rules R1–R9, ADR-001/002, and the Inspector + Reporting protocols** are the
  governance contract for the Flutter app too (CLAUDE.md says "R1–R9 חלים על שני
  הפרויקטים"). R2 (no window) is absolute.

**Adapt (platform-specific):**
- `barcode.ts` / `voice.ts` are Web-API wrappers (BarcodeDetector, Web Speech). In
  Flutter use native plugins (mobile_scanner / speech_to_text); keep the Hebrew
  fallback error strings and `he-IL` locale.
- The DOM `data-*` mirroring effect in `app-settings.ts` has no equivalent — apply
  theme/text-size/contrast through Flutter's theming instead, but keep the same
  setter API + persistence.

**Frozen / deprecated — DO NOT copy as-is:**
- **`IMPLEMENTATION_PROTOCOL.md`** is DEPRECATED (R2 violation) — historical only.
- The **6 dashboard docs + ROLE_DRAWER_SYSTEM + UI_ARCHITECTURE** describe full
  windows; they are *content references*, not layout targets. Every screen/drawer/
  overlay/sheet they describe must become a **dial drill** (the BS dial drill pattern
  from Phases F–H is the template). The 3 reverts are the cautionary tale: do not
  build a persona dashboard as a full `<main>`/route — ever.
- `MenuTab` no longer has `'catalog'` (moved to search FAB, INSP-0044). The menu FAB
  is **4 tabs** (home/projects/cart/settings); the search FAB is **5 tools** (voice/

- The invented `search-filters` / `search-recent` features (and `SortMode`) are
  scaffolding disclosed in INSP-0002 — port only if the owner still wants them.
