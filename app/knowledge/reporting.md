# Reporting Protocol — דיווח לבעלים

> אחרי כל פעולה משמעותית (commit, ביקורת, החלטה, גילוי), Claude חייב
> לתת סיכום בעברית בפורמט הקבוע למטה. **אסור לסטות מהפורמט.**

## מתי לדווח (חובה)

- אחרי `git commit` (סיכום מה נכנס)
- אחרי הרצת מפקח שמחזיר NO-GO או stuck-loop
- אחרי גילוי בעיה שדורשת החלטה מהבעלים
- בסוף סדרת שינויים (5+ קבצים)
- כשנדרשת בחירה ממה שאני מציע

לא חובה לדווח על:
- שאלות בירור פנימיות
- בדיקות-משנה זמניות
- typecheck / build רוטיניים שעוברים בלי הפתעה

## פורמט קבוע

```
## ✅ סיכום — {שם הפעולה בעברית}

**מה עשיתי**
{משפט-שניים בעברית פשוטה, בלי ז'רגון. למשל: "הוספתי 2 ADRs ו-spec.json עם 29 features"}

**איך הלך**
{תוצאת הביקורת: ✓ GO או ✗ NO-GO + מספרי חומרה}
- CRITICAL: 0
- MAJOR:    0
- MINOR:    2 (תיקנתי אחד, השני מתועד)

**שינויים שאתה רואה**
- {bullet 1: דבר שהמשתמש רואה במסך/בקבצים}
- {bullet 2: ...}
- {bullet 3 לכל היותר}

**סיכונים**
{אם יש: דבר שיכול להישבר/לדרוש תשומת לב. אם אין: "אין סיכון מהותי"}

**מצב נוכחי**
{מספרים מהותיים: X/Y מיופים, Z בדיקות, וכו'. שורה אחת}

**מה הלאה**
1. {אופציה א}
2. {אופציה ב}
3. {אופציה ג} (לכל היותר 3)

**צריך החלטה ממך?**
{שאלה ספציפית או "לא — אני ממשיך"}
```

## כללי כתיבה

1. **עברית פשוטה**. בלי שמות פונקציות באמצע משפט אלא אם הכרחי. בלי
   "scaffold", "stub", "API" — תרגם.
2. **מספרים, לא ערפול**. "19/29" עדיף על "רוב הפיצ'רים".
3. **קצר**. סיכום שלם — מקסימום 200 מילים. אם ארוך יותר — כיווץ.
4. **פעולה**. כל סיכום מסתיים בשאלה או בהצעה — לא ב"זהו".
5. **סיכונים תמיד**. אסור "אין סיכון" אם יש משהו ידוע. אם באמת
   אין — לכתוב "אין סיכון מהותי" במפורש.

## דוגמה טובה (קצרה)

```
## ✅ סיכום — Phase 1 פרוטוקול ידע

**מה עשיתי**
הוספתי 2 ADRs (החלטות אדריכלות שתועדו רטרואקטיבית),
legacy-map עם 30 מיפויים, ו-spec.json עם 29 features וסטטוס לכל אחד.

**איך הלך**
✓ GO. המפקח מצא 2 MINOR (חוסר עקביות בקובץ JSON, ופיצ'ר אחד
שצוין כ"המצאה" עם תיעוד). תיקנתי את הראשון.

**שינויים שאתה רואה**
- 4 קבצים חדשים תחת `app/knowledge/`
- ADR-001 ו-ADR-002 מנמקים החלטות שדיברנו עליהן בשיחה
- spec.json מראה: 14 features מוכנים, 2 חלקיים, 13 חסרים

**סיכונים**
שני features (search-filters, search-recent) הומצאו ולא קיימים
באב-טיפוס. תיעדתי אותם בשקיפות. אם תרצה להסיר — נסיר.

**מצב נוכחי**
19/29 features מוכנים או חלקיים. שני דוחות מפקח נשמרו לארכיון.

**מה הלאה**
1. בניית features חסרים (cart, manager dashboard מלא, ...)
2. Phase 2 של הפרוטוקול (contracts/, Spec Auditor)
3. שילוב — בניית persona אחד + spec.json מתעדכן בזמן אמת

**צריך החלטה ממך?**
איזה מסלול לבחור?
```
