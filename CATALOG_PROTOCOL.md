# BuildSmart · פרוטוקול ידע — קטלוג מוצרים

מסמך-על שמכסה את **כל** מערכת הקטלוג מקצה לקצה: מבנה הנתונים, המקורות,
תהליך ה-QA, האימות מול המקור, וההתאמה (תאימות) — כדי לתחזק עשרות אלפי
מוצרים בלי לאבד שליטה.

---

## 1. מבני הנתונים (4 קבצים)

| קובץ | מה מחזיק | יחידה |
|------|----------|-------|
| `data/lipskey_catalog.dart` | **כל המוצרים** (כ-788, יגדל) | `LipskeyCatalogProduct` |
| `data/catalog_tree.dart` | עץ הצלילה (קטגוריה→תת→עלה) | `CatalogNode` |
| `data/smart_tree.dart` | כרטיסי "מוצר חכם" (בחירת מותג+אביזרים) | `SmartProduct` |
| `data/brands.dart` | מותגים/יצרנים | `Brand` |

### LipskeyCatalogProduct — שדות
`sku` (מק"ט, מזהה ייחודי) · `nameHe` · `nameEn` · `categoryHe` · `categoryEn`
· `categoryEmoji` · `page` (עמוד במקור) · `imageFile` (תמונה או null) · `brand`
(AQUATEC / ליפסקי) · `color` · `qtyPack` · `qtyPallet` · `dims`

נגזרות: `imageAsset` · `specImageAsset` (crop עמוד PDF) · `typeEmoji`
(אייקון לפי סוג כשאין תמונה).

### הקשר בין הקבצים
```
CatalogNode(leaf).lipskeyCategory ─┬─→ מסנן את LipskeyCatalogProduct לפי categoryHe
                                    └─→ רשימת המוצרים בעלה
CatalogNode(leaf).smartKey ───────────→ SmartProduct (fallback למוצרי grohe/hamat)
LipskeyCatalogProduct.brand ──────────→ Brand (aquatec/lipskey)
```

## 2. המקורות (PDFs)

- ליפסקי הוא **מפיץ**, לא יצרן יחיד. הקטלוג מכיל **2 מותגים**:
  - **ליפסקי** (~255): ניקוז, סיפונים, מחסומים, אסלות, מיכלי הדחה, צינורות.
  - **AQUATEC** (~601): ברזים, מקלחות, נחושת, HDPE, NTM, חבקים, גינה.
- מקור הנתונים = **4 PDFs**. כרגע בידינו רק קטלוג הניקוז של ליפסקי (30 עמ').
- **לכיסוי אימות מלא צריך את כל 4 ה-PDFs.** בלעדיהם ~89% מהמוצרים
  לא ניתנים לאימות סמנטי.

## 3. כלי ה-QA — `scripts/catalog_qa.py`

| פקודה | מה עושה |
|-------|---------|
| `audit` | בדיקה **תחבירית**: פגום/ריק/כפול-מק"ט/שם-כפול/ללא-גודל/ללא-תמונה/לא-ממופה |
| `export <csv>` | CSV מלא עם דגלי-בעיה לכל מוצר (סינון ב-Excel) |
| `crosscheck <pdf...>` | בדיקה **סמנטית**: כל שם מול טקסט ה-PDF ליד המק"ט; מדווח % כיסוי + חשודים |
| `pdfmap <pdf>` | מיפוי עמוד→מק"טים |
| `verify <pdf> sku...` | רינדור עמודי המקור ל-PNG לקריאה ויזואלית ותיקון מדויק |

**שתי שכבות בדיקה — שתיהן חובה:**
1. **תחבירית** (`audit`) — תופסת זבל, כפילויות, שדות חסרים. ✅ עוברת.
2. **סמנטית** (`crosscheck`) — תופסת שם *שגוי* (נקי אבל לא נכון).
   דורשת את ה-PDFs. ⚠️ כיסוי נוכחי 11% (PDF אחד).

> **שם "נקי" ≠ שם נכון.** רק crosscheck מול המקור מוכיח נכונות.

## 4. תהליך עבודה (כל סבב QA)

```
1. python3 scripts/catalog_qa.py audit                 # שגיאות תחביר
2. python3 scripts/catalog_qa.py crosscheck <כל ה-PDFs> # שגיאות סמנטיקה
3. python3 scripts/catalog_qa.py export /tmp/qa.csv     # סקירה מלאה
4. לכל חשוד:  verify <pdf> <sku>  → קריאה ויזואלית → תיקון שם
5. flutter analyze && flutter build web --release       # אימות קוד
6. commit + push
```

### כלל הזהב
**שם שלא ניתן לאמת מול המקור — לא ממציאים.** מותר רק:
- ניקוי בטוח (הסרת זבל/אריזה, נרמול ״→") — בלי שינוי משמעות.
- שם שנקרא ויזואלית מעמוד ה-PDF (`verify`).

## 5. תיקוני נתונים בטוחים (Python, על `nameHe` בלבד)

- נרמול גרשיים: `״ “ ”` → `"` (להשאיר `׳` לגיטימי כמו ג׳קוזי).
- הסרת מקטעי אריזה: `"NN כמות באריזה/במשטח"`.
- פיצול שדות מאוחדים `|` — לאחד רק את החלקים התיאוריים.
- רווח בגבול ספרה↔אות (`1.5מחסום`→`1.5 מחסום`) **בלי לשנות סדר**.
- ⚠️ **לעולם לא** להכניס `'` בודד למחרוזת single-quote ב-Dart (שובר קומפילציה).
  שמות עם גרש → לעטוף ב-double-quote `"..."`.

---

## 6. תאימות — "מה מתחבר למה" (לב האפליקציה)

- כל מוצר נושא **מידות DN** בשם (DN50, 50/40, 130/50, 3/4").
- בכרטיס המוצר: סקשן **"חיבורים תואמים"** מפצל לפי **צד** (קצה):
  - מצמד 75/50 → צד DN75 + צד DN50, כל אחד עם החלקים התואמים מקטגוריות אחרות.
- התאמה = חיתוך קבוצות-מידה מנורמלות (מתעלם מכמות/אורך).
- **לכן `NOSIZE` חשוב**: מוצר בלי DN בשם → לא משתתף בתאימות.
  השלמת גדלים מה-PDF = שיפור התאימות.

## 7. תמונות וזיהוי

- מוצר עם `imageFile` → תמונת מוצר אמיתית.
- בלי תמונה → `typeEmoji` (אייקון לפי סוג: ניפל🔩 מצמד🔌 צינור📏 ...) +
  שם תיאורי. **לא** אימוג'י קטגוריה אחיד (אי-אפשר להבחין).
- crop עמוד PDF (`specImageAsset`) זמין דרך כפתור "פרטים/מפרט" + הגדלה.
- האגנט משלים `imageFile` ל-200 המוצרים החסרים (פורמט `<sku>.jpg` →
  `assets/lipskey/products/`). כשמתווסף — ה-fallback נסוג אוטומטית.

## 9. הוספת קטלוג/מותג חדש (סקלביליות)

1. הוסף את ה-PDF; הרץ extractor (`scripts/extract_*.py`) → רשומות.
2. הוסף `Brand` ב-`brands.dart`.
3. הוסף `categoryHe` חדשות; מפה אותן לעלים ב-`catalog_tree.dart`
   (`audit` יתריע על קטגוריה לא-ממופה).
4. הרץ `audit` + `crosscheck <pdf החדש>` → תקן.
5. אותו פרוטוקול בדיוק — לא משנה אם 800 או 80,000 מוצרים.
