# BuildSmart · Catalog Engine — תוכנית פעולה (100 צעדים, 8 גלים)

מטרה: להפוך את הקטלוג ממסמך+סקריפט ל**מערכת אכיפה אוטומטית** שמבטיחה אפס
שגיאות גם ב-80,000 מוצרים. כל גל = ערך עצמאי. סטטוס: ⬜ פתוח · 🔄 בעבודה · ✅ הושלם.

---

## גל 1 — מנוע ולידציה (engine) 🔄
מ-`audit` פשוט ל-מנוע חוקים עם חומרה (severity) והוכחת-עצמו.
1. ✅ מודל severity: ERROR (חוסם) · WARN · INFO.
2. ✅ ~15 חוקים: ריק · זבל · מספר-תקוע · סוגריים · כפול-מק"ט · שדות-חסרים ·
   קטגוריה-לא-ממופה · ללא-גודל · ללא-תמונה · nameEn-זבל · brand-לא-חוקי ·
   קטגוריה↔סוג (מילון) · גודל-תקין · אורך-שם · כפילות-שם.
3. ✅ `selftest` — fixtures "מורעלים" שמוכיחים שכל חוק יורה (עונה ל"הכלי לא עובד").
4. ✅ פלט JSON (`--json`) לצריכה ע"י CI/כלים.
5. ⬜ דירוג חומרה per-category בדוח.

## גל 2 — נרמול ותיקון אוטומטי 🔄
6. ✅ `fix` — נרמולים בטוחים אידמפוטנטיים (גרשיים, זבל-אריזה, רווח ספרה↔אות).
7. ✅ הגנת גרש-Dart (single→double quote אוטומטית).
8. ⬜ `fix --dry-run` (ברירת מחדל) מול `--apply`.
9. ⬜ לוג שינויים (diff) לכל תיקון.
10. ⬜ נרמול `nameEn` (הסרת `|`).

## גל 3 — מקור-אמת ואימות 100% ⬜
11. ⬜ סכמת `source_truth.json` + ולידטור סכמה.
12. ⬜ `truthcheck` מלא: name/category/size/brand diff + חסר/עודף.
13. ⬜ `truthapply` — תיקון אוטומטי מהמקור (עם dry-run).
14. ⬜ אימות תמונה↔מק"ט (hash/קיום קובץ).
15. ⬜ דוח כיסוי-אימות (% מאומת מול מקור).

## גל 4 — אכיפת CI (gate) 🔄
16. ✅ GitHub Action: `catalog_qa.py audit --json` חוסם merge על ERROR.
17. ⬜ pre-commit hook מקומי.
18. ⬜ באנר-סטטוס ב-README (badge).
19. ⬜ דוח QA כ-artifact ב-CI.

## גל 5 — מנוע תאימות מוקשח ⬜
